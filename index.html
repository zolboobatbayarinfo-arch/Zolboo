SME Data Visibility — LendMN-ий 4 paths, and their subpaths. Not a catalog of options — the foundation of strategic choice.

§ 01 — Core Logic
Data + Engine = Outcome

Engine хэр хүчтэй байх нь Data хэр баялаг байхаас хамаарна. Иймд өнөөдрийн шийдвэр бол: Data-г хэрхэн үүсгэх вэ. Доор танилцуулсан 4 зам нь энэ нэг асуултын дөрвөн өөр хариулт.

§ 04 — Sequencing, not Selection

Which path goes first?

Эдгээр замууд mutually exclusive биш. Асуулт нь "аль замаар явах вэ" биш, "аль нь урдаа орж явах вэ". Portfolio approach — дараах sequencing санал болгож байна:

Now — 9 months
Phase A · Speed + Quality
Path 3 base + Path 4b

Хурдан coverage (embedded API)-г tier-1 bank data quality (realtime statement)-тэй хослуулж, data foundation-оо эхэлж байгуулна. Capital-light, break-even ~1,500 merchant.

9 — 18 months
Phase B · Anchor Rail Decision
Path 2 (Bonum / Negdi)

PSP анкерийн шийдвэр — Bonum-тай 2a minority → 2c JV-ийн хувиарлах. Path 3 sub-paths-ыг өргөжүүлнэ (ТЕГ, POS, logistics).

18 — 36 months
Phase C · Regulatory & Scale
Path 4e + Path 4d

Open Banking advocacy-р regulatory infrastructure-г хэлбэржүүлэх. Bank white-label channel-аар distribution өргөтгөх (шинэ segment).

3+ years · Optionality
Phase D · Own Rails (optional)
Path 1 (if strategic)

Phase A–C амжилттай болсон тохиолдолд Path 1-г авч үзнэ. Энэ нь starting point биш, graduation destination.

§ 06 — What these paths do NOT include
Data + Engine = Outcome

Эдгээр 4 зам бол Data-г үүсгэх замууд.

Engine — буюу Aegis — бол data-г process хийж outcome гаргах давхарга. Aegis нь data visibility үүсгэх хэрэглүүр биш. Board эдгээр замуудыг үнэлэхдээ Aegis-ээс тусад нь үнэлэх ёстой — Aegis өөрөө хэнтэй ажиллахаас нь үл хамааран эдгээр замын аль нэгэн дээр суурилна.

→ Data path decision today. Aegis work continues in parallel.

§ 07 — The Question Before the Board

What is LendMN:
rail operator, consolidator, orchestrator, or bank intelligence layer?

Зам тус бүр LendMN-ий identity-г өөрчилнө. Board энэ identity answer-ийг зөвшөөрч байж execution sequencing-г сонгож чадна. Cost model-ын нарийвчилсан тоо дараагийн хувилбарт сайжирна — өнөөдрийн шийдвэрлэх зүйл бол аль замыг эхлээд эхлүүлэх вэ гэдэг асуулт.
